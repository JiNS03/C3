# 자유 주제 자동화 설계 및 구현
### 코디세이 3번째 미션 - 프로젝트 2

---

## 1. 자동화할 반복 업무 정의

**반복 업무**: 매일 여러 통씩 들어오는 이메일(문의/알림 메일 등)을 하나씩 열어 내용을 확인하고, 핵심 내용을 정리해서 요약해두는 작업

**문제 상황**: 메일 한 통 확인하고 요약하는 데 몇 분이 걸린다고 해도, 하루에 여러 통이 쌓이면 반복 작업만으로 상당한 시간이 소모됨. 특히 "긴급"으로 분류되어야 할 메일과 일반 메일을 구분해서 대응 속도를 다르게 가져가야 하는 점도 매번 사람이 판단해야 하는 번거로움이 있음.

**자동화 목표**: 새 이메일이 도착하면, 제목에 "긴급" 포함 여부로 자동 분류하고, AI가 본문을 요약한 뒤, 분류에 맞는 제목을 붙여 요약 메일이 자동 발송되도록 한다.

---

## 2. 도구 선정 및 선정 이유

**선정 도구**: Make

**선정 이유**:
- 프로젝트 1에서 이미 Make로 Router(조건 분기)와 Google 서비스 연동을 다뤄본 경험이 있어, 동일한 환경에서 AI 모듈까지 확장하기 수월함
- Make는 Google Gemini AI 모듈이 기본 내장되어 있어, 별도의 복잡한 연동 설정 없이 "AI 요약 Action"을 추가할 수 있음
- Gemini는 무료 등급(Free tier) API Key 발급이 가능해, 소규모 테스트 용도로는 별도 비용 없이 사용 가능함 (과금 리스크 완화)
- Gmail은 이미 Google 계정 연결이 되어 있어 별도의 워크스페이스/채널 설정 없이 바로 발송 Action을 구성할 수 있어 가장 간단함
- 모듈 단위로 **"Add error handler"** 기능을 붙일 수 있어, 실패 알림 및 대체 경로(보너스2) 설계가 구조적으로 명확함

## 3. 워크플로우 설계

### 전체 구조

```
Trigger: Gmail – Watch emails (라벨 "요약대상" 붙은 메일 감지)
     ↓
조건 분기(Router): 제목에 "긴급" 포함 여부
     ├─ 긴급   ─ Action 1: Google Gemini AI – 본문 3줄 요약 생성
     │           Action 2: Gmail – "[긴급] 메일 요약" 발송
     └─ 긴급 아님 ─ Action 1: Google Gemini AI – 본문 3줄 요약 생성
                    Action 2: Gmail – "[일반] 메일 요약" 발송

[보너스2, 계획] 각 Gmail 발송 Action 실패 시
     → 에러 핸들러: 담당자에게 별도 실패 알림 메일 발송 + 백업 시트에 임시 기록
```

### 단계별 흐름 설명

1. **Trigger** — Gmail에서 **"요약대상"** 라벨이 붙은 메일을 `Watch emails` 모듈(Gmail 필터, Query: `label:요약대상`)로 감지
2. **조건 분기(Router)** — 감지된 메일의 제목(Subject)에 **"긴급"**이라는 단어가 포함되어 있는지 여부로 두 갈래로 분기
   - **1st 경로(긴급)**: `Subject` Contains `긴급`
   - **2nd 경로(일반)**: `Subject` Does not contain `긴급`
3. **Action 1 – AI 요약 [보너스1]** — 각 경로에서 **Google Gemini AI**(Generate a response) 모듈을 통해 메일 본문(Full text body)을 3줄로 요약하는 프롬프트를 실행
4. **Action 2 – Gmail 발송** — 발신자, 원본 제목, AI 요약 결과를 조합한 내용을 담아 지정한 수신자에게 이메일 발송
   - 긴급 경로: 제목 `[긴급] 메일 요약`
   - 일반 경로: 제목 `[일반] 메일 요약`
5. **에러 핸들러 [보너스2, 계획]** — Gmail 발송 단계에서 오류가 발생할 경우, 시나리오가 멈추지 않고 계속 진행되도록 "Resume" 처리하면서, 담당자에게 별도의 실패 알림 메일을 보내고, 실패한 메일 정보를 백업용 스프레드시트에 임시로 기록하여 데이터 유실을 방지 (다음 단계로 구현 예정)

- **Trigger**: 1개 (Gmail Watch emails)
- **Action**: 2개 (Google Gemini AI 요약, Gmail 발송)
- **조건 분기**: 1개 (제목 내 "긴급" 포함 여부, Router 사용)
- 긴급/일반 두 경로 모두 실제 테스트 메일로 각 1회 이상 실행하여 정상 동작 확인 완료

### 자동 실행 구조

Make 시나리오 하단의 스케줄 토글을 **ON**으로 설정하여, Gmail Trigger가 주기적으로 새 메일을 확인하고 조건에 맞으면 자동으로 전체 흐름이 실행되도록 구성 예정. (Trigger 발생 시 사람의 개입 없이 자동 실행되도록 마무리 작업 진행 중)

## 4. 구현 및 실행 결과

### 구현 과정 요약

1. Gmail **Watch emails** Trigger 추가 → Filter type을 "Gmail 필터"로 변경 후 Query에 `label:요약대상` 입력하여, 요약 대상으로 라벨링된 메일만 감지하도록 구성
2. **Router** 추가 → 제목의 "긴급" 포함 여부로 1st(긴급)/2nd(일반) 두 경로로 분기
3. 각 경로에 **Google Gemini AI(Generate a response)** 모듈 추가 → API Key 연결 후, 메일 본문을 3줄 요약하는 프롬프트 구성
4. 각 경로에 **Gmail(Send an email)** 모듈 추가 → Body type을 "Raw HTML"로 설정하고, 발신자/원본 제목/AI 요약 결과를 조합한 내용을 작성
5. 테스트 메일 2건(긴급 1건, 일반 1건)을 직접 작성해 발송하고 "요약대상" 라벨을 붙인 뒤, Trigger의 "Choose where to start"를 "All"로 설정하여 기존 메일부터 처리되도록 조정
6. **Run once**로 전체 흐름 실행 → 두 경로 모두 정상적으로 AI 요약 생성 및 메일 발송까지 완료됨을 확인

## 5. 보너스 과제 반영 내역

### 보너스 1 – AI 연동 Action 추가 (완료)
워크플로우의 Action 1 단계에서 **Google Gemini AI** 모듈(Generate a response)을 사용해, 감지된 메일의 본문을 3줄로 요약하는 텍스트 생성 Action을 구현함. 긴급/일반 두 경로 모두 각각 Gemini 모듈을 연결하여, 분기된 이후에도 AI 요약이 정상 동작함을 확인함.

> 참고: Google Gemini API는 무료 등급(Free tier)으로 발급 및 사용이 가능해, 별도의 유료 결제 없이 보너스 과제를 완수할 수 있었음.

### 보너스 2 – 실패 알림 및 재시도 전략 (진행 예정)
Gmail 발송 Action에 에러 핸들러를 연결하여, 발송 실패 시:
1. 담당자에게 별도의 이메일로 실패 알림 발송
2. 실패한 메일 정보를 백업 스프레드시트에 임시 기록 (대체 경로)
3. 에러 핸들러 유형을 "Resume"으로 설정하여 시나리오 전체가 중단되지 않도록 처리

위 구조로 설계는 완료했으며, 실제 Make 시나리오에 에러 핸들러 모듈을 추가하는 구현은 다음 단계로 진행 예정.
